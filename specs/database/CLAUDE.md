# Database Specifications - Claude Guide

## Purpose
Database specs define data models, schemas, and persistence strategies.

## Files in This Directory
- `schema.md` - Complete database schema with all tables
- `migrations.md` - Migration strategy and version history

## How to Use
1. Read schema before creating models
2. Follow naming conventions
3. Add indexes as specified
4. Use migrations for schema changes
5. Update spec when schema evolves

## Creating a New Database Spec
Use the template: `@.spec-kit/prompts/database-spec-template.md`
